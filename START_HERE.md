
```powershell

```

### Restart Application
```powershell
ssh -i "$PWD\bmk-key.pem" ubuntu@<EC2_IP> `
    "cd /opt/bmk/deploy && docker compose restart backend"
```


## 🎯 Common Questions

### Q: What if I already have AWS infrastructure?

**A:** No problem! You can:
1. Choose "Option 2" to create just what's missing
2. Edit `.env.aws` manually with your existing endpoints
3. Run "Option 4" to just deploy to existing EC2

### Q: Can I change the region?

**A:** Yes! The scripts accept a `-Region` parameter:
```powershell
& deploy-aws.ps1 -Region "us-east-1"
```

### Q: What if the scripts fail midway?

**A:** 
1. Don't worry - infrastructure is partially created but stopped
2. You can re-run the same script
3. It will detect existing resources and skip them
4. Or fix the issue and resume

### Q: How do I stop paying AWS charges?

**A:** Delete resources:
```powershell
# Delete EC2 instance
aws ec2 terminate-instances --instance-ids i-xxxxx --region ap-south-1

# Delete RDS database
aws rds delete-db-instance --db-instance-identifier bmk-postgres `
    --skip-final-snapshot --region ap-south-1

# Delete ECR repository
aws ecr delete-repository --repository-name bmk-backend --force --region ap-south-1

# Delete security groups (after instances are terminated)
aws ec2 delete-security-group --group-id sg-xxxxx --region ap-south-1
```

### Q: How do I access the database from my machine?

**A:** 
```powershell
# Install PostgreSQL client if needed, then:
psql -h <RDS_ENDPOINT> -U bmk_user -d busymumkitchen

# Password is in your DEPLOYMENT_SUMMARY.txt
```

## 📞 Troubleshooting Tips

### If script fails with "AWS CLI not found"
```powershell
# Reinstall AWS CLI v2 from official source
# Or run in admin PowerShell to install:
Start-Process -FilePath 'msiexec.exe' `
    -ArgumentList '/i', 'https://awscli.amazonaws.com/AWSCLIV2.msi', '/quiet' `
    -Wait
```

### If containers won't start
```powershell
ssh -i "$PWD\bmk-key.pem" ubuntu@<EC2_IP>
cd /opt/bmk/deploy
docker compose logs backend --tail 50
# Check for DB connection errors, missing env variables, etc.
```

### If you can't SSH to EC2
```powershell
# Check key file permissions
icacls "$PWD\bmk-key.pem"

# Fix if needed (run as admin):
icacls "$PWD\bmk-key.pem" /inheritance:r
icacls "$PWD\bmk-key.pem" /grant:r "%USERNAME%:F"
```
